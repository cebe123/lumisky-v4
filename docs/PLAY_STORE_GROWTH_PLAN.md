# Lumisky Play Store Growth Plan

Prepared for `com.lumisky.android` / Lumisky.

## Research Snapshot

- Official Play guidance prioritizes clear descriptions, accurate screenshots, compliant metadata, strong preview assets, custom listings, and store listing experiments.
- Android vitals affect Play visibility. Keep user-perceived crash rate, ANR rate, wake locks, and battery behavior below bad-behavior thresholds.
- Competitor wording in live wallpaper results repeatedly uses: `live wallpaper`, `4K`, `HD`, `AMOLED`, `3D/4D`, `parallax`, `battery-friendly`, `sky`, `moon`, `stars`, `clouds`.
- Lumisky should not compete on fake catalog volume. Position it as a real-time sky live wallpaper with sun/moon timing, smooth shader rendering, and battery-aware modes.

## Positioning

Primary promise:

> A calm real-time sky live wallpaper that follows the sun, moon, and time of day.

Differentiators:

- Real-time sun/moon/daylight behavior, not only looping video.
- Battery-aware live wallpaper rendering.
- Sky, moon, parallax, and quality/performance controls.
- Clean wallpaper setup flow with location or city-based timing.

Avoid claims:

- `#1`, `best`, `top`, `millions`, `download now`, unverifiable battery promises, or features not currently in the app.

## Main Store Listing Draft

App name candidates, keep one:

1. `Lumisky Live Wallpaper`
2. `Lumisky Sky Wallpaper`
3. `Lumisky`

Recommended short description:

> Real-time sky live wallpaper with sun, moon, parallax, and battery modes.

Alternative short descriptions:

- `Dynamic sky live wallpaper with sun, moon, and smooth parallax.`
- `Battery-aware sky live wallpaper with sun and moon timing.`

Long description draft:

```text
Lumisky is a dynamic live wallpaper for Android, designed around a calm sky that changes with time.

Set a living sky on your home screen with sun position, moon timing, parallax motion, and performance controls. Lumisky uses shader-based rendering instead of a simple looping video, so the wallpaper can feel alive while staying practical for everyday use.

Key features:
- Dynamic sky live wallpaper
- Sun, moon, sunrise, sunset, and celestial timing
- Smooth parallax effect
- Quality and performance modes
- Battery-aware rendering options
- Works from the Android live wallpaper picker

Use Lumisky if you want a clean sky, nature, or AMOLED-friendly live wallpaper that changes through the day without turning your home screen into a noisy animation.

Notes:
- Live wallpaper availability can depend on device manufacturer support.
- Location can be used to calculate local sun and moon timing.
- You can choose performance settings based on your device and battery preference.
```

## Preview Asset Brief

Icon:

- 512x512 PNG, no ranking/price text.
- Visual idea: crescent moon + sun horizon + deep sky gradient. Keep center-safe.

Feature graphic:

- Show actual Lumisky sky wallpaper on a phone home screen.
- Text max 3-5 words: `Real-time sky wallpaper`.
- No badges like `best`, `top`, `free`, `new`.

Screenshots, first 6:

1. Home screen with live sky applied: `Real-time sky`.
2. Sunrise/day/night comparison: `Changes with time`.
3. Sun and moon timing/settings: `Sun and moon sync`.
4. Parallax or motion preview: `Smooth parallax`.
5. Performance mode screen: `Balanced for battery`.
6. Wallpaper picker/set flow: `Easy setup`.

Video, 15-25 seconds:

- 0-4s: phone home screen with animated sky.
- 4-9s: transition from sunrise to night.
- 9-14s: moon/parallax close-up.
- 14-20s: settings/performance mode.
- End frame: Lumisky icon + `Real-time sky wallpaper`.

## Custom Store Listings

Create 3 listings after the main listing is stable:

- TR listing: Turkish text and Turkish screenshots.

- Sky/moon listing: emphasize real-time sun/moon timing, sunrise/sunset, stars/clouds if present.

## Experiment Plan

Run one variable at a time:

1. Short description: `real-time sky` vs `battery-aware sky`.
2. First screenshot: real home screen vs time comparison.
3. Feature graphic: phone mockup vs full-bleed wallpaper visual.
4. App name: `Lumisky` vs `Lumisky Live Wallpaper` if title limit and brand strategy allow.

Decision metric:

- Primary: store listing conversion rate.
- Secondary: uninstall rate after install, crash/ANR vitals, review text.

## 14-Day Execution Plan

Day 1-2:

- Confirm current feature truth: live wallpaper, location/city timing, parallax, performance modes.
- Finalize English and Turkish metadata.

Day 3-5:

- Capture real screenshots on one modern phone and one smaller/older device.
- Produce feature graphic and 15-25s preview video.

Day 6:

- Update main listing and TR localization.
- Check policy-sensitive wording and Data safety alignment.

Day 7-10:

- Release to internal/closed testing if needed.
- Watch crashes, ANRs, battery complaints, wallpaper picker failures.

Day 11-14:

- Start first store listing experiment.
- Reply to early reviews with short, specific answers.
- Create issue list from reviews and vitals, fix only high-impact problems first.

## Quality Gate Before Promotion

- `.\gradlew.bat :app:validatePlayReleaseConfig`
- `.\gradlew.bat :app:lintDebugLocal`
- `.\gradlew.bat :app:validateShaderCelestialMotionContinuity`
- Manual smoke test: set wallpaper, return home, lock/unlock, switch performance mode, rotate/resize if supported.
- Play Console: confirm no active crash/ANR/battery bad-behavior warning.
